第 1 章 学习之初

Linux 是什么

  • Linux 其实是一个操作系统平台
  • 互联网大公司的服务器 99% 的操作系统都是 Linux


Linux 的由来

  • Richard Stallman 发起了GNU 计划
  • GPL 协议：GNU 计划的所有软件都必须自由使用、自由更改、自由发布。也就是说，发布软件必须要发布它的源代码，这个源代码可以供别人自由使用，可以被随便更改，但是必须要发布更改后的代码
  • GNU 计划发起后，这期间产生了许多非常棒的软件，比如 vi、Emacs、GCC
  • 1991 年，芬兰大学生林纳斯·本纳第克特·托瓦兹（Linus Benedict Torvalds）基于兴趣开发了一个类 Unix 操作系统，它就是 Linux
  • 1994 年，Linux 加入 GNU 计划并采用 GPL 协议发布

Linux 怎么读

  • 官方给出的标准发音为['li:nэks]，写成中文就是“李呢克斯”

常见 Linux 发行版

  • Debian（1993）
  • Slackware（1993）
  • Red Hat（1994）
    • 2002年推出 Red Hat Enterprise（RHEL）
    • Red Flag（1999）
    • Fedora
    • CentOS（诞生于 2003 年），2014 年 2 月，CentOS 被 Red Hat 收入囊中
  • Ubuntu（2004）[在 Debian 的基础上发展起来的，Ubuntu 只是 Debian 的一个分支]
  • SUSE（1994）[Slackware的分支]
  • Android 操作系统也是一种 Linux 发行版

我们要学习哪个 Linux 发行版

  • 国内大多数企业使用 RHEL/CentOS 作为服务器操作系统
  • RHEL 是 Red Hat 公司推出的一款针对企业的发行版 Linux，可以免费下载使用。但是要想获得官方授权，就必须要购买授权协议（也就是所谓的服务）
  • RHEL 在没有获得授权的情况下不能使用 yum 工具
  • CentOS 有免费的 yum 工具可以使用
  • 推荐以后使用 CentOS 发行版来学习 Linux
    • 国内大多数企业使用 RHEL 搭建服务器
    • 目前使用 CentOS 的企业越来越多
    • CentOS 和 RHEL 几乎一样，而且 CentOS 有免费的 yum 工具可以使用
    • CentOS 目前已经加入 Red Hat 公司，且依然完全免费

并非强制日后一定要使用 CentOS，其实所有版本的 Linux 都大同小异，只要学会了其中一个，学其他发行版自然是水到渠成的事



Linux 系统管理员要养成的习惯

要习惯使用命令行

  • 早期的 Linux 并不完全支持图形界面
  • Linux 的图形界面存在许多小问题，所以未被大多数 PC 机用户认可
  • 开启图形界面不仅耗费资源，而且远程管理时还会有网络带宽的额外开销，因此 Linux 在服务器领域比较流行
  • 命令行是 Linux 系统正常运行的核心，也是专业 Linux 系统工程师必须掌握的技能，所以我们要习惯使用命令行

操作要严谨

  • 务必养成备份数据的好习惯
  • 输入命令的速度不要太快，看准了再按回车
  • 对于重要的配置文件，在修改前一定要进行备份

安全不可忽视

  • 安全使用密码：
    • 密码最好不要长期沿用（建议每隔 1~3 个月修改一次）
    • 密码不能保存在一个文档里、不能把存有密码的文档存到可移动存储设备
    • 员工的计算机一定要设置好密码，并且员工在离开工位时要把计算机锁屏
    • 借助一款工具 KeePass 来记忆密码


学习建议

稳中求进

  • 技术方面的书，光看一遍可不行，我们的目的是要学会和掌握技能
  • 建议每章内容都花 2~3 天的时间来学习，其中的小案例需要多练习才能够真正掌握

善于总结文档

  • 善于总结文档，以便日后举一反三

复习很关键

  • 记完笔记并不等于掌握了知识，要想完全掌握，就必须经常复习

举一反三

  • 需要掌握的并不是小案例本身，而是小案例背后的知识

课后习题

(1) 请查一查 Linux 的发展历史，并列举几种有代表性的 Linux 发行版。

(2) 请简述 GNU 和 GPL 两个概念。

(3) 列举几种基于 GPL 协议发行的软件。

(4) 列举几个比较流行的 Linux 版本，并说一说它们的特点
